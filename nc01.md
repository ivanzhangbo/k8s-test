# ✅ 2. **テスト時の実際準備コマンド（手順込み）**

ここでは Linux 上に **ダミー SRC / DEST ディレクトリを作ってテストできる環境を準備するコマンド** を提示します。

---

# 🔧 **1) ダミーディレクトリ作成**

```bash
mkdir -p /tmp/src1 /tmp/src2
mkdir -p /tmp/dest1 /tmp/dest2
```

---

# 🔧 **2) ダミーファイル作成**

```bash
echo "test1" > /tmp/src1/file1.txt
echo "test2" > /tmp/src2/file2.txt
```

---

# 🔧 **3) SSH 接続の代わりにローカル rsync 用ショートカット作成**

rsync の SSH モードをローカルに合わせるため mock を作ります。

```bash
ln -s /usr/bin/ssh /root/.ssh/id_rsa
```

（※テストでは本当の SSH は不要）

または DEST をローカルパスにして SSH を使わないようにすることも可：

```
/tmp/dest1/
/tmp/dest2/
```

---

# 🔧 **4) CSV テストファイル作成**

```
/tmp/src1,/tmp/dest1
/tmp/src2,/tmp/dest2
```

保存先は：

```bash
nano /etc/websync_list.csv
```

---

# 🔧 **5) スクリプト実行**

```bash
bash websync.sh
```

---

# 🔧 **6) 同期結果確認**

```bash
ls -l /tmp/dest1
ls -l /tmp/dest2
```

---

# 🔧 **7) 更新テスト（更新検知）**

```bash
echo "update" >> /tmp/src1/file1.txt
bash websync.sh
cat /tmp/dest1/file1.txt
```

---

# 🔧 **8) 削除テスト（--delete の動作）**

```bash
rm /tmp/src1/file1.txt
bash websync.sh
ls /tmp/dest1   # → file1.txt が消えていれば OK
```

---

# 🔧 **9) ロックテスト**

```bash
touch /root/websync.lock
bash websync.sh  # → スキップされること
```

ロック解除：

```bash
rm /root/websync.lock
```

---

# 🔧 **10) 異常系テスト**

### ● SRC が存在しない

```
/notfound,/tmp/dest1
```

### ● DEST が存在しない

```
/tmp/src1,/notfound
```

### ● CSV の壊れた行

```
/tmp/src1
```

これらを CSV に追加してテスト。
